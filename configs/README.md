# 配置文件说明

`configs` 目录用于存放不同硬件/场景下的整机配置。根据所用机型、摄像头与算力选择对应的 YAML。

| 文件名 | 适用场景 | 主要特点 |
| --- | --- | --- |
| `example.yaml` | 通用示例 | 同时给出 USB 与工业相机参数、完整自瞄与标定字段，便于快速参考与复用。 |
| `demo.yaml` | 演示/低算力 | CPU 推理 + 传统方法联合，开启基本 ROI，包含 planner 与 buff 调参，适合演示或调试。 |
| `camera.yaml` | 相机基础模板 | 仅包含常用工业相机 (`hikrobot`) 的曝光/增益等基础设置，可作为其他配置的相机子集。 |
| `calibration.yaml` | 标定流程 | 棋盘格规格、云台-IMU 初始矩阵及相机曝光等，用于相机/云台标定阶段。 |
| `ascento.yaml` | Ascento 平台 | 工业相机 + GPU 推理，含云台/IMU 标定矩阵、自瞄与 buff 参数，默认敌方颜色为蓝。 |
| `standard3.yaml` | 标准三代 | 工业相机 + GPU，自瞄/跟踪/规划全量参数，含标定数据与 buff 检测模型路径。 |
| `standard4.yaml` | 标准四代 | 工业相机 + GPU，提供云台与标定参数，适配标准四代底盘/云台。 |
| `sentry.yaml` | 哨兵位 | 工业相机 + GPU，自瞄与 buff 支持，包含云台串口与 CAN 配置，默认敌方为红。 |
| `mvs.yaml` | MindVision 方案 | 面向 MindVision 工业相机的配置，包含 USB 备用参数，GPU 推理与 buff 支持，默认敌方为蓝。 |
| `uav.yaml` | 空中/轻量平台 | MindVision 工业相机 + 高帧率 USB 备用，CPU 推理，偏向轻量快速响应的自瞄参数。 |

命名含义：同名字段在各文件间保持一致（如 `enemy_color`、`device`、相机曝光、标定矩阵、buff 检测/射击参数等），可在对应文件上调整以适配具体机型。使用时按需要将目标 YAML 路径传给启动命令或在程序配置中引用。
